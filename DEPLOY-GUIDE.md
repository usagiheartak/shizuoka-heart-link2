## ステップ1️⃣：GitHubアカウントを作る（5分）

### 1-1. GitHubのウェブサイトを開く
ブラウザ（Chrome、Safariなど）で以下のURLを開きます：
```
https://github.com
```

### 1-2. アカウント作成
1. 右上の「Sign up」ボタンをクリック
2. メールアドレスを入力
3. パスワードを決めて入力（8文字以上）
4. ユーザー名を決めて入力
   - 例: `shizuoka-ishi` や `sasaki-heart` など
   - ⚠️ このユーザー名が公開URLに使われます！
5. 「Create account」をクリック
6. メールに届いた認証コードを入力

**既にアカウントを持っている場合:**
右上の「Sign in」からログインしてください。

---

## ステップ2️⃣：新しいリポジトリを作る（3分）

**「リポジトリ」とは？**
→ ファイルを保管する「箱」のようなものです。

### 2-1. リポジトリの作成画面を開く
1. ログイン後、右上の「+」ボタンをクリック
2. 「New repository」を選択

### 2-2. リポジトリの設定
以下の項目を入力します：

**Repository name（リポジトリ名）:**
```
shizuoka-heart-link
```
⚠️ 正確に入力してください（スペースや大文字小文字に注意）

**Description（説明）- 任意:**
```
静岡心不全連携システム（しずおかハートリンク）
```

**Public / Private:**
→ **「Public」を選択**
（誰でもアクセスできるようにするため）

**Add a README file:**
→ ☑️ チェックを**入れない**
（すでにREADME.mdがあるため）

### 2-3. リポジトリを作成
画面下の緑色ボタン「Create repository」をクリック！

---

## ステップ3️⃣：ファイルをアップロードする（10分）

### 3-1. アップロード画面を開く
新しく作成されたリポジトリのページで、
**「uploading an existing file」**というリンクをクリック

### 3-2. ファイルを選択
#### 📌 重要：正しい選び方

**❌ NG：フォルダごとドラッグしない**
フォルダ全体（shizuoka-heart-link）をそのままドラッグしてはいけません

**⭕ OK：フォルダの「中身」を選ぶ**

1. `shizuoka-heart-link`フォルダを開く
2. 中にある**全てのファイルとフォルダ**を選択
   - index.html
   - sheet-A.html
   - sheet-B.html
   - README.md
   - images フォルダ
   - downloads フォルダ
   
3. 選択した全てを、GitHubの画面にドラッグ&ドロップ

### 3-3. アップロード確認
画面に以下のようなリストが表示されればOK：
```
✓ index.html
✓ sheet-A.html
✓ sheet-B.html
✓ README.md
✓ images/flow-diagram.png
✓ images/reimbursement-diagram.png
✓ downloads/sheet-A.xlsx
✓ downloads/sheet-B.xlsx
```

### 3-4. コミット（保存）
画面下にスクロールして：
1. 「Commit changes」と書かれた緑色のボタンをクリック
2. 数秒〜1分ほど待つ
3. リポジトリのメインページに戻ります

---

## ステップ4️⃣：GitHub Pagesを有効にする（5分）

### 4-1. 設定画面を開く
1. リポジトリのページ上部にある「**Settings**」タブをクリック
2. 左側のメニューから「**Pages**」を選択

### 4-2. GitHub Pagesの設定

**Source（ソース）:**
- 「Deploy from a branch」が選ばれていることを確認

**Branch（ブランチ）:**
- ドロップダウンメニューから「**main**」を選択
- フォルダは「**/ (root)**」を選択
- 「**Save**」ボタンをクリック

### 4-3. 公開URLの確認
1. 約1〜2分待つ
2. ページを再読み込み（F5キーまたはブラウザの更新ボタン）
3. 画面上部に緑色の枠で以下のように表示されます：

```
✅ Your site is live at https://ユーザー名.github.io/shizuoka-heart-link/
```

このURLをクリックすると、公開されたウェブサイトが見られます！

## 🔧 トラブルシューティング

### Q1. 「404 Page not found」と表示される
**原因:** ファイルの配置が間違っているか、GitHub Pagesが有効になっていない

**解決方法:**
1. Settings → Pagesで「main」ブランチが選ばれているか確認
2. リポジトリのトップに`index.html`があるか確認
   - フォルダの中にあってはダメ！トップ階層に配置

### Q2. 画像が表示されない（□アイコンになる）
**原因:** `images`フォルダがアップロードされていない

**解決方法:**
1. リポジトリのファイル一覧で`images`フォルダがあるか確認
2. なければ、ステップ3をやり直す
3. 画像ファイルだけ個別にアップロードしてもOK

### Q3. Excelファイルがダウンロードできない
**原因:** `downloads`フォルダがアップロードされていない

**解決方法:**
1. リポジトリのファイル一覧で`downloads`フォルダがあるか確認
2. なければ、ステップ3をやり直す

### Q4. スマホで見ると表示が崩れる
**原因:** ブラウザのキャッシュが古い

**解決方法:**
1. スマホのブラウザでページを開く
2. 画面を下にスワイプして更新（リロード）
3. それでもダメなら、ブラウザのキャッシュをクリア

### Q5. URLを変更したい
**解決方法:**
1. Settings → 「General」
2. 「Repository name」を変更
3. 「Rename」をクリック
4. URLが `https://ユーザー名.github.io/新しい名前/` に変わります

## 🔄 ファイルを更新する方法

後日、内容を変更したい場合：

### 方法1: ブラウザから直接編集（簡単）
1. GitHubのリポジトリページを開く
2. 編集したいファイル（例：index.html）をクリック
3. 右上の鉛筆アイコン（✏️）をクリック
4. 内容を編集
5. 画面下の「Commit changes」をクリック
6. 約1分後、ウェブサイトに反映されます

### 方法2: ファイルを差し替える
1. リポジトリページで該当ファイルをクリック
2. 「Delete file」で削除
3. 「Add file」→「Upload files」で新しいファイルをアップロード
